👤 Usuario

  • Titular de las tarjetas. Datos demográficos, geográficos y financieros.
  • credit_score como indicador base de riesgo
  • Coordenadas para detección de anomalías geográficas
  • Ratio yearly_income / total_debt como feature de solvencia

💳 Tarjeta

  • Instrumento financiero asociado a un Usuario
  • card_on_dark_web → feature crítica para el modelo
  • has_chip correlaciona con seguridad de la transacción
  • card_type prepaid asociado a mayor riesgo estadístico

💸 Transaccion

  • Evento central del sistema, evaluado por el modelo ML
  • errors (PIN incorrecto, etc.) como señal de fraude
  • use_chip indica si se usó chip o banda magnética
  • merchant_city permite detectar transacciones geográficamente imposibles

🏪 CodigoMCC

  • Catálogo de ~90 categorías de comercios (Merchant Category Code)
  • Permite agrupar y detectar patrones por sector
  • Alto riesgo: casinos, money transfer, prepaid cards

🚨 AlertaFraude

  • Notificación accionable cuando el score supera un umbral
  • severity: LOW / MEDIUM / HIGH / CRITICAL
  • status: OPEN → REVIEWING → CONFIRMED / DISMISSED
  • Relación 0..1 con Transaccion (no toda genera alerta)

🧠 PrediccionFraude

  • Resultado del modelo ML sobre cada transacción (1:1)
  • fraud_score en [0,1] para umbrales configurables
  • Trazabilidad: qué versión del modelo generó la predicción

⚙️ ModeloML

  • Registro de versiones del modelo de detección
  • Almacena métricas: precision, recall, AUC-ROC
  • is_active identifica el modelo en producción activo

📊 PerfilRiesgo

  • Entidad derivada que agrega comportamiento histórico del usuario
  • Se recalcula periódicamente (batch o event-driven)
  • risk_score compuesto usado como feature del modelo
