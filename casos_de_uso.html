Descripción de Casos de Uso

ID Actor(es) Nombre Descripción Relaciones
UC-01 Usuario Iniciar sesión El usuario se autentica con credenciales para acceder al sistema. —
UC-02 Usuario Cerrar sesión El usuario finaliza su sesión activa de forma segura. —
UC-03 Analista Ver dashboard Visualización de KPIs en tiempo real: volumen de transacciones, alertas activas, tasa de fraude del día. —
UC-04 Analista Buscar transacción El analista busca una transacción por ID, tarjeta, usuario o comercio. «include» UC-06
UC-05 Analista Ver detalle transacción Consulta completa de una transacción: datos del comercio, tarjeta, usuario y predicción de fraude asociada. «include» UC-13
UC-06 Analista Filtrar transacciones Aplicar filtros por rango de fecha, monto mínimo/máximo, categoría MCC, tipo de tarjeta o estado de alerta. «include» en UC-04
UC-07 Analista Ver alertas Listado de alertas de fraude ordenadas por severidad, con filtros de estado (OPEN/REVIEWING/CLOSED). «extend» desde UC-14
UC-08 Analista Revisar alerta El analista examina el detalle de una alerta: transacción sospechosa, score de fraude, contexto del usuario. «include» UC-09
UC-09 Analista Confirmar / Desestimar El analista cierra la alerta marcándola como fraude confirmado o falso positivo. Alimenta el feedback loop del modelo. «include» en UC-08
UC-10 Analista Generar reporte Generación de reportes periódicos de fraude agrupados por MCC, región geográfica, tipo de tarjeta o usuario. —
UC-11 Analista Ver métricas modelo Consulta de métricas de performance del modelo activo: precision, recall, F1, AUC-ROC y evolución histórica. «include» UC-15
UC-12 Analista Exportar datos Descarga de datasets filtrados en formato CSV o JSON para análisis externo. —
UC-13 Data Scientist Motor ML Ejecutar predicción Pipeline de inferencia: recibe una transacción y retorna un fraud score junto a los features que más influyeron. «include» UC-14
UC-14 Motor ML Calcular fraud score Cálculo del score [0,1] para una transacción usando el modelo activo. Si supera el umbral, extiende UC-07. «include» en UC-13 · «extend» UC-07
UC-15 Data Scientist Evaluar modelo Corre métricas de evaluación sobre el conjunto de test. Genera reporte de precision/recall/AUC-ROC por versión. «include» en UC-11
UC-16 Data Scientist Reentrenar modelo Lanza el pipeline de entrenamiento con nuevos datos etiquetados, incluyendo feedback del analista (UC-09). «include» UC-17
UC-17 Data Scientist Activar versión modelo Pone en producción una nueva versión del modelo tras validación. Desactiva la versión anterior. «include» en UC-16
UC-18 Administrador Gestionar usuarios Alta, baja y modificación de usuarios del sistema, asignación de roles (Analista, Data Scientist, Admin). —
UC-19 Administrador Configurar umbrales Ajuste del umbral de fraud score a partir del cual se genera una alerta, y configuración de niveles de severidad. —
