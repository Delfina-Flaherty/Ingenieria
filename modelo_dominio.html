👤 Usuario

Titular de las tarjetas. Contiene datos demográficos, geográficos y financieros que permiten construir un perfil de comportamiento.

  • credit_score como indicador de riesgo base
  • Coordenadas para detección de anomalías geográficas
  • yearly_income / total_debt como ratio de solvencia

💳 Tarjeta

Instrumento financiero asociado a un usuario. El campo card_on_dark_web es una feature crítica para el modelo de fraude.

  • has_chip correlaciona con seguridad de la transacción
  • card_type (prepaid) asociado a mayor riesgo
  • credit_limit como umbral de detección de montos anómalos

💸 Transacción

Evento central del sistema. Cada transacción es evaluada por el modelo para asignarle un score de fraude.

  • Errores en la transacción (PIN incorrecto, etc.) como señal
  • use_chip indica si se usó chip o banda magnética
  • merchant_city permite detectar transacciones geográficamente imposibles

🏪 CodigoMCC

Catálogo de categorías de comercios (Merchant Category Code). Permite agrupar transacciones por rubro y detectar patrones de fraude por sector.

  • ~90 categorías disponibles en el dataset
  • Categorías de alto riesgo: casinos, money transfer, prepaid

🧠 PrediccionFraude

Resultado del modelo ML sobre cada transacción. Almacena el score continuo y la clasificación binaria final.

  • fraud_score en rango [0, 1] para umbrales ajustables
  • Trazabilidad: qué modelo y versión generó cada predicción

🚨 AlertaFraude

Notificación accionable generada cuando una predicción supera un umbral. Tiene ciclo de vida propio (pendiente → revisada → resuelta).

  • severity: LOW / MEDIUM / HIGH / CRITICAL
  • status: OPEN / REVIEWING / CONFIRMED / DISMISSED

⚙️ ModeloML

Registro de versiones del modelo de detección. Permite comparar performance entre versiones y hacer rollback.

  • Almacena métricas: precision, recall, AUC-ROC
  • is_active indica el modelo en producción

📊 PerfilRiesgo

Entidad derivada que agrega comportamiento histórico del usuario. Se recalcula periódicamente para reflejar cambios de patrón.

  • Frecuencia y monto promedio de transacciones
  • Historial de fraudes confirmados
  • risk_score compuesto como feature del modelo
